# PhoneSystem-Web Architecture

## Overview

A web-based 8-line phone system for broadcast production, replacing the PyQt5 desktop application with a Flask + HTML/JS/CSS solution. Designed for Raspberry Pi with touchscreen, but accessible from any device on the network.

## API Endpoints

### Lines API

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/lines` | Get status of all 8 lines |
| GET | `/api/lines/<id>` | Get status of specific line |
| POST | `/api/lines/<id>/dial` | Dial a number on line |
| POST | `/api/lines/<id>/hangup` | Hang up line |
| POST | `/api/lines/<id>/channel` | Set audio output channel |

### Configuration API

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/config/sip` | Get SIP configuration |
| POST | `/api/config/sip` | Save SIP configuration |
| GET | `/api/config/network` | Get network configuration |
| POST | `/api/config/network` | Save network configuration |

### Audio API

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/audio/test` | Start test tone on channel |
| DELETE | `/api/audio/test` | Stop test tone |

### System API

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/system/status` | Get system health status |
| POST | `/api/system/restart` | Restart phone system |
| POST | `/api/system/reboot` | Reboot Raspberry Pi |

## WebSocket Events

Real-time updates via Socket.IO:

### Server â†’ Client

| Event | Data | Description |
|-------|------|-------------|
| `line_status` | `{line_id, state, caller_id, duration}` | Line state changed |
| `sip_registration` | `{line_id, registered, error}` | SIP registration update |
| `audio_level` | `{line_id, level}` | Audio level meter |

### Client â†’ Server

| Event | Data | Description |
|-------|------|-------------|
| `subscribe` | `{lines: [1,2,3...]}` | Subscribe to line updates |

## Data Models

### Line State

```json
{
  "line_id": 1,
  "state": "idle",           // idle, ringing, active, on_hold
  "phone_number": "",
  "caller_id": "",
  "duration": 0,
  "audio_channel": 0,        // 0=None, 1-8=Output channel
  "sip_registered": true,
  "muted": false
}
```

### SIP Configuration (Multi-Account)

```json
{
  "mode": "multi_account",
  "sip_server": "sip.twilio.com",
  "sip_port": 5060,
  "transport": "UDP",
  "num_lines": 8,
  "lines": [
    {
      "line_id": 1,
      "username": "+1XXXXXXXXXX",
      "password": "xxxxxx",
      "caller_id_name": "Studio Line 1",
      "caller_id_number": "+1XXXXXXXXXX"
    }
    // ... lines 2-8
  ]
}
```

### Network Configuration

```json
{
  "mode": "static",          // "dhcp" or "static"
  "ip_address": "192.168.1.221",
  "subnet_mask": "255.255.255.0",
  "gateway": "192.168.1.1",
  "dns_server": "8.8.8.8"
}
```

## Technology Stack

### Backend
- **Python 3.9+** - Core language
- **Flask** - Web framework
- **Flask-SocketIO** - WebSocket support
- **baresip** - SIP user agent
- **ALSA** - Audio routing

### Frontend
- **HTML5** - Structure
- **CSS3** - Styling (custom, no framework)
- **JavaScript (ES6)** - Interactivity
- **Socket.IO Client** - Real-time updates

### Deployment
- **systemd** - Service management
- **Chromium** - Kiosk browser
- **netplan/dhcpcd** - Network configuration

---

## Hardware Requirements

| Component | Model | Purpose |
|-----------|-------|---------|
| Computer | Raspberry Pi 4 (4GB+) | Main system |
| Display | WIMAXIT 12" Touchscreen | 1366x768 UI |
| Audio | Focusrite Scarlett 8i6 | 8-channel output |
| Network | Ethernet (recommended) | SIP/RTP traffic |

## Installation

```bash
# Clone repository
git clone https://github.com/omaralabed/PhoneSystem-Web.git
cd PhoneSystem-Web

# Run installation script
./scripts/install.sh

# Start service
sudo systemctl start phonesystem
sudo systemctl enable phonesystem
```

## Migration from PyQt5

This project replaces the previous PyQt5 desktop application with a web-based solution:

| Old (PyQt5) | New (Web) | Benefit |
|-------------|-----------|---------|
| `main_window.py` | `templates/index.html` + `static/js/main.js` | Multi-device access |
| `SIPSettingsDialog` | `/sip-settings` page | No freeze issues |
| `VirtualKeyboard` | Native browser keyboard | Works everywhere |
| Modal dialogs | Separate pages | More stable |
| Custom styling (QSS) | CSS | Standard, easier to maintain |
